# AlleyPin 簡報設計規範

AlleyPin 品牌簡報 = **深藍主調 + 幾何色塊 + 橘色點綴**，乾淨、專業、留白足。
用 HTML 當製作原稿（1280×720 / 16:9），再轉成 PPTX 或 Google 簡報。

**本 skill 的範圍 = AlleyPin 的「品牌視覺」與「書寫規範」**：色盤、字體規則、封面、內容頁框架、logo、文案寫法。**適用於任何主題**的 AlleyPin 簡報（提案、產品介紹、報告、教育訓練…皆可）。

**不在範圍內 → 請發揮你的簡報專業自行判斷，本 skill 不限制你**：內容結構與敘事、版面配置、一頁放幾張卡/怎麼排、資訊層次、要不要用圖表。把畫面做好、把故事說好是你的本職；本 skill 只負責「讓它看起來、讀起來是 AlleyPin」。不要為了套版而犧牲表達。

## 數字校準（動手前先做）

簡報的數字若**來源之間衝突、或不確定**（例如 brief 寫一個數、附件 / PDF 寫另一個），**先跟使用者求證對齊，再拿它做計算或衍生數字**——絕不可自己挑一個「讓算式好看」的數字硬算。
當下無法確認時：用 ⚑ 標註待確認、或**直接做兩個版本**讓使用者最後挑選 / 修改。

## 內容架構（動手排版前就要想）

每頁的關鍵訊息，**先想「最有力的呈現方式是什麼」，再排版**——不是把字丟上去再美化。
- 有沒有能講故事的**呈現裝置**？例如**算式**（小投入 ÷ 比率 = 大產出）、左右對比、before / after、流程、趨勢、大數字 callout。
- **數字放大、中文說明縮小**，拉開層次，視線一眼落在重點。

這是內容架構的判斷（你的本職），**在第一版就要想**；用標準色，裝置只是工具、別僵化硬套。

## 版面與留白（產出時就要顧，不是事後補）

留白要「主動」不要「空洞」：
- **主動留白**：刻意的呼吸空間，讓主角更突出 → 好。
- **被動空洞**：內容擠成幾座孤島、其餘大片空著，看起來沒做完 / 內容太小 → 不好。

**第一版製作時就要把它考慮進去，不是事後才補。** 做每一頁當下自問：主角夠大嗎？左右、上下平衡嗎？空白有沒有在工作？若某半邊整片空，就把主角放大、或用支撐視覺（圖表、品牌幾何）讓空白工作。

**這是判斷原則，不是固定數值或公式**——每頁依內容拿捏，別僵化套版、也別追求填滿；適度的留白本身就是好設計。

## 第一步（必做）：先問輸出格式 → 決定標題字體

開始做之前，**先問使用者：「最終要 PPTX 離線檔，還是 Google 簡報？」** 這決定標題字體：

| 輸出 | 標題字體 | 原因 |
|---|---|---|
| **Google 簡報** | `Noto Sans TC` | LINE Seed 不在 Google 字體庫，加不進去、會被替換 |
| **PPTX 離線檔** | `LINE Seed TW`（標題）+ `Noto Sans TC`（內文） | LINE Seed 為開源（OFL）。**但**：開檔電腦需已安裝該字體、或把字體嵌入 pptx；且此 pptx 一旦匯入 Google 簡報，LINE Seed 仍會掉 |

內文一律 `Noto Sans TC`。**模板標題預設 `Noto Sans TC`（Google 簡報安全、零修改）**；若輸出 PPTX 離線檔且本機已安裝 LINE Seed，再把標題字體改成 `"LINE Seed TW","Noto Sans TC"`。

## 字級

## 封面（兩款，擇一）

兩款封面結構都是「左側放文字、右側放幾何」，深藍底。完整幾何座標見模板，直接複製改字即可：

- **封面 A — 右側幾何 motif**（`templates/cover-a.html`）：右緣一排幾何（橘三角→白條→淺藍圓→深藍半圓→橘半圓），等比撐滿上下、貼齊右緣。較簡潔通用。
- **封面 B — 全藍幾何拼塊**（`templates/cover-b.html`）：右半由 `#1C3667` 直欄＋橫條、主藍半圓拱、底部淺藍三角＋`#1C3667` 直欄組成，整組貼齊上/下/右緣。較有設計感。

**標題排版慣例**（兩種都可，寫進選項）：
1. 大標在橘斜線**上方**、次標在斜線**下方**（斜線當分隔）— 範本正規做法。
2. 斜線在上，大標＋次標都放斜線下方 — 亦可。

左側所有物件（斜線、大標、次標、logo）**一律齊左 96px**。斜線用 SVG 畫（最左點落在 96px），不要用旋轉方塊（旋轉後左角會凸出、視覺不齊）。

## 內容頁框架（固定，`templates/content-page.html`）

- 底色 `#EDEEF3`
- 左上小矩形 `#1D4A82`：**貼齊最上緣**、左邊距 64px
- 底部細藍 bar `#1D4A82`：左側 `Noto Sans TC` 白字「AlleyPin」、右下白色頁碼
- **頁尾兩種**：(1) 基本 = 藍條（如上）；(2) 聯絡頁尾（公版 p.62）= 藍條上方靠右再加一條 `www.alleypin.com/tw ｜ @alleypin ｜ +886 2 2745 7177`（細直線分隔、官網字為連結），適合封底 / 聯絡頁
- 中間內容區**完全自由**：版面、卡片數、排法依內容與你的簡報專業判斷，**沿用本色盤即可**（白底卡片＋頂部色條只是示意，非規定）

## 書寫規範（產出文案一律遵守）

1. 中文字、中文標點一律**全形**。
2. 唯獨**括號用半形 `( )`、刪節號用半形 `...`**；半形括號前後加**半形空格**。
3. 英文、數字、英文標點一律**半形**；英文標點後加半形空格。
4. 中英文接續處加**半形空格**。
5. 品牌名：`AlleyPin`、`AlleyPin Nova`（產品全名 `AlleyPin Nova 企業卡`）、`PinMed`（兩字不空格、首字大寫）、`DBC`（全大寫）、`1.Talk`。
6. 第三方用對外全稱：`LINE`（非 Line）、`Facebook`（非 FB）、`Visa`（非 VISA）、`遠東商業銀行`（非 遠東商銀）。**全稱不確定時先問使用者，勿臆測。**
7. 概數：成數用**國字**（如「五成」）；百分比用**半形數字**（如 `35%`）。
8. 金額：圓整大額用「N 萬」（半形數字＋國字單位，如 `10 萬`、`100 萬`），最利簡報掃讀；精確金額用完整數字（如 `NT$5,000`）。

範例：`AlleyPin 提供診所最便利的 LINE CRM 服務，串接全台超過 11 家病歷系統商 ( HIS )，支持全台超過九成的診所營運。`

## 封底 / Thank You 頁（`templates/closing-page.html`）

官方收尾頁（參考公版 p.62）：
- 左：**白色 logo 置中於深藍矩形塊**（lockup）。
- 右：收尾標題（如 Thank You）＋ 聯絡人（圓形頭像位＋姓名＋職稱＋Email）。右側整塊**上緣對齊 lockup 上緣、下緣對齊 lockup 下緣（等高）**。
- 底：聯絡頁尾。**不放**左上小矩形。

## 圖示與選用元件（選用，非必用）

- **圖示風格**：簡單線條 icon，放在品牌色圓角方塊內（icon 白色）。**AlleyPin 官方 icon 庫在公版簡報 p.70–72**，優先取用，勿混入其他風格。
- **時間軸 / 流程圖等元件**：用品牌色（軸線淺灰、節點主藍、進度條 主藍 / 過渡藍 / 橘）。用得上就用——但「要不要用、怎麼排」是你的簡報判斷，本 skill 不規定。

## 產出與轉檔

1. 複製對應模板**連同 skill 的 `assets/` 資料夾**到你的簡報資料夾（logo 路徑用 `assets/...`）→ 填字（遵守書寫規範）→ 內容版面依主題自由設計（**範例的大數字卡只適合數字型內容；非數字主題請改用清單/步驟/圖文等合適版面，別硬湊數字**）→ 在瀏覽器確認 1280×720。
2. 轉 Google 簡報：把 HTML 另存 / 匯入（或用 Chat App 的「open in Drive」），標題記得已是 `Noto Sans TC`。
3. 轉 PPTX：可用 pptxgenjs 依模板座標重建；要用 LINE Seed 須確保字體已安裝或嵌入。

## 收尾：交付前自己先看過（必做）

每頁產出後，當成 1280×720 實際看一遍再交付，至少檢查：
- **對齊**：數字與標籤、左右 / 上下元素有沒有對齊？（尤其字級不同的元素，注意底線對齊）
- **重疊 / 超界**：元素有沒有疊在一起、超出邊界、文字溢出？
- **層次**：主角夠突出嗎？視線落點對嗎？

沒看過、沒檢查的稿不要交。

## 常見錯誤

- 標題用 LINE Seed 卻輸出 Google 簡報 → 字會掉。先問輸出格式。
- 斜線用旋轉方塊 → 左緣不齊。用 SVG 線。
- 左上矩形留了上邊距 → 要貼齊最上緣。
- 文案寫 `VISA`/`FB`/全形括號 → 違反書寫規範。
- logo 不顯示 → 忘了把 `assets/` 一起複製、或 src 路徑沒指對。
- 為了套用範例版面而硬湊內容（如非數字主題硬塞大數字）→ 版面服務內容，不是反過來。
