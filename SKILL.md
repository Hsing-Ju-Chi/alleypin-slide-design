# AlleyPin 簡報設計規範
*最後更新：2026.7.9*

AlleyPin 品牌簡報 = **深藍主調 + 幾何色塊 + 橘色點綴**，乾淨、專業、留白足。
用 HTML 當製作原稿（1280×720 / 16:9），再轉成 PPTX、Google 簡報或 Keynote。

**本 skill 管**：AlleyPin 的「品牌視覺」＋「書寫規範」（色盤、字體、封面、內容頁框架、封底、logo、文案）＋ 幾條製作心法。適用**任何主題**。
**本 skill 不管（你的簡報專業自行判斷）**：內容結構與敘事、版面配置、一頁放幾張卡 / 怎麼排、要不要用圖表。把畫面做好、把故事說好是你的本職；別為套版犧牲表達。
> 固定的是**品牌對齊基準**（封面齊左 96px、內容頁齊左 64px、框架元素照模板）；自由的是**內容區放什麼、怎麼排**。

## 製作流程（一份簡報怎麼走）

0. **先問輸出格式**：PPTX 離線檔 / Google 簡報 / Keynote？→ 決定標題字體
1. **數字校準**：來源數字有衝突先求證
2. **內容架構**：先想最有力的呈現方式
3. **版面與留白**：第一版就顧平衡
4. **套品牌**：色盤 / 字體 / 封面 / 內容頁框架 / 封底 / logo / 書寫規範
5. **收尾自檢**：對齊 / 重疊 / 層次

> 1–3、5 是**判斷心法（非僵化、第一版就要做）**；4 是**品牌固定規範**。

## 製作心法（判斷原則，非套公式）

- **數字校準**：數字若來源衝突 / 不確定（brief 一個數、附件 / PDF 另一個），**先跟使用者求證，再拿來計算或衍生**——別挑「讓算式好看」的數字硬算。無法當下確認就 ⚑ 標註、或做兩版讓使用者挑。
- **內容架構**（動手排版前就想）：關鍵訊息先想「最有力的呈現方式」——
  - 有沒有講故事的**呈現裝置**？算式（小投入 ÷ 比率 = 大產出）、左右對比、before / after、流程、趨勢、大數字 callout。
  - **數字放大、中文說明縮小**，拉開層次，視線一眼落在重點。
  - **多階段 / 路線圖**做成循序漸進、朝願景前進的故事（命名階段、可加 GO / NO-GO Gate），給人明確願景想像 = 專業感。
- **版型服務內容——不硬套，也不為新而新**：參考版型（卡片 / 流程鏈 / 對比 / 關係圖…）是**可靠的預設，多數頁用它們剛剛好**（一致、快、專業）。重點不是「避免用模板」，而是**先看內容本質再決定形式**：現成版型常常就是最貼切的選擇；**只有當內容明顯有更好的呈現時**，才為那一兩頁另做新版面——不是每頁都要求新。**別每頁硬套同一招，也別為了新奇而破壞好用的東西。** 守住品牌骨架（色盤 / 字體 / 框架 / 主動留白）的前提下，選**最合適**的，不是最新奇的。
  - **但跳脫要誠實，別冒充真實介面**：不要做「仿真實產品 UI」的假 mockup（假 LINE 對話、假 IG 畫面、假系統截圖感）——與真實差距一大就顯得假、不專業。新樣子可以**抽象化、原創、資訊圖表化**，但別假裝是真的畫面。
- **版面與留白**：留白要「**主動**」（呼吸空間、襯托主角），別「**被動空洞**」（內容擠成孤島、其餘大片空著像沒做完）。**大氣** = 少用框、多用留白與細線分隔；內容夠份量時，滿版橫向 row 常比並排小卡更從容。某半邊整片空 → 放大主角、或加支撐視覺（圖表 / 品牌幾何）。
- **收尾自檢**（交付前必做）：把每頁當 1280×720 看一遍——**對齊**（尤其不同字級的底線）、**重疊 / 超界 / 溢出**、**層次**、**圖片 / logo 沒被拉伸變形（比例＝原圖）**、**封面右側幾何沒跑版（照「封面」段**該款封面**的幾何驗收逐項對——A 是一直條、B 是右半整片拼塊，標準不同）**。沒看過的稿不要交。**而且要在「實際交付的那個格式」上看**——出 Google 簡報就真的上傳 Drive 開 Google 簡報看一遍、出 Keynote 就開 .key 看一遍，別只看 HTML 或某一種格式的預覽就當作全部都對（轉檔後 logo 比例、字體、位置偶爾會跑）。

## 改寫 / 重繪既有簡報（rebrand）

把別人的簡報改成 AlleyPin 版時：**內容保留、視覺換品牌**，不是重寫內容。
1. **讀出原檔內容**：`.key` / `.pptx` → 用 Keynote 匯出 PDF（看版面）＋ pptx（讀文字）；或解 pptx 的 slide XML 抽 `<a:t>` 文字。
2. **逐頁判斷版型**（封面 / 卡片 / 流程鏈 / 對比 / 表格 / 關係圖 / 分隔 / 收尾），用品牌文法重排——**原本的深色 / 他牌視覺全部丟掉**。
3. **忠於原意**：facts、數字、專有名詞照舊（濃縮措辭 OK，**別無中生有**；不確定的數字先問，見數字校準）。
4. 照「產出與轉檔」出可編輯檔、逐頁收尾自檢。

## 輸出格式 → 標題字體

開始做之前先問使用者：「最終要 PPTX 離線檔、Google 簡報，還是 Keynote？」

| 輸出 | 標題字體 | 原因 |
|---|---|---|
| **Google 簡報** | `Noto Sans TC` | LINE Seed 不在 Google 字體庫，加不進去、會被替換 |
| **PPTX 離線檔** | `LINE Seed TW`（標題）+ `Noto Sans TC`（內文） | LINE Seed 為開源（OFL），但開檔電腦需已裝該字體或把字體嵌入；此 pptx 一旦匯入 Google 簡報，LINE Seed 仍會掉 |
| **Keynote**（Mac） | 同離線 PPTX：`LINE Seed TW`（標題）+ `Noto Sans TC`（內文） | Keynote 是本機軟體，**本機有裝 LINE Seed 就正確顯示**；沒裝則用 Noto。 |

內文一律 `Noto Sans TC`。**模板標題預設 `Noto Sans TC`（Google 簡報安全、零修改）**；PPTX 離線 / Keynote 且本機裝了 LINE Seed，才把標題字體改成這個 stack：
`"LINE Seed TW_OTF","LINE Seed TW_TTF","LINE Seed TW","Noto Sans TC"`
**⚠️ macOS 實際安裝的家族名多帶後綴（`LINE Seed TW_OTF` / `LINE Seed TW_TTF`）——只寫 `LINE Seed TW` 會對不上、靜默 fallback 到 Noto。** 家族名以你機器為準，用 `system_profiler SPFontsDataType | grep -i "line seed"` 查。沒裝就維持 Noto，別硬塞。

**圖表配色**：主序列 = 主藍；次序列 = 過渡藍 / 淺藍；強調・對比 = 橘 / 深橘；正向 = 綠；軸線・格線 = 淺灰 `#D7DCE4`。
**低對比提醒**：重要文字要跟背景有足夠明暗差（深字配淺底、白字配深底）。**淺藍 / 淺橘 / 淺綠只當色塊或大字強調，別當小內文**（淺色小字壓在淺底會糊）。

## 字級與行距（規範值＝Keynote 檢閱器顯示的 pt；HTML 模板用 px，換算 px = pt × ⅔）

> 校準來源：2026-07-09 依《脂肪肝 AI-PRM 年會演講版》（公版 1920×1080 畫布）定案，以 Keynote 呈現為主。

| 元素 | Keynote pt | HTML px |
|---|---|---|
| 封面大標 | 75–78 | 50–52 |
| 內容頁主標題 | **48** | 32 |
| 內容頁副標題 | **36** | 24 |
| 敘述內文・卡片說明（**中文下限**） | **≥24** | ≥16 |
| eyebrow（主標上方橘色英文小亮點）・標籤 | 20 | 13.5 |
| 大數字 | 57–96（粗體、tabular） | 38–64 |
| 頁碼・免責聲明・來源出處・聯絡頁尾 | 不受下限限制（chrome，維持模板值） | — |

**中文文案不得小於 24pt（16px）**——演講投影距離下看不清。唯免責聲明、來源出處、頁碼等 chrome 例外。

**行距（敘述多行文字）**：Keynote / pptxgenjs 的行距倍數 **1.1–1.3**（1.5 太鬆）。
**⚠️ CSS 的 `line-height` 數字不能直接抄進 pptxgenjs 的 `lineSpacingMultiple`**——兩者定義不同，Noto Sans TC 實測 **Keynote 倍數 ≈ CSS ÷ 1.46**（CSS 1.6–1.9 ≈ Keynote 1.1–1.3）。HTML 模板寫 `line-height:1.6–1.75`；重建 pptx 時 `lineSpacingMultiple` 直接給 1.1–1.3，別搬 CSS 值（搬了 Keynote 會顯示 1.5+，就是「行距虛胖」的來源）。

## 封面（兩款，擇一）

兩款都是「左側放文字、右側放幾何」，深藍底。完整幾何座標見模板，複製改字即可：
- **封面 A — 右側幾何 motif**（`templates/cover-a.html`）：右緣一排幾何（橘三角→白條→淺藍圓→深藍半圓→橘半圓），等比撐滿上下、貼齊右緣。較簡潔通用。
- **封面 B — 全藍幾何拼塊**（`templates/cover-b.html`）：右半（x=763 起）整片拼塊——中欄 `#1C3667` 上下兩大塊夾四條橫條；右上淺藍底＋主藍半圓拱（拱頂觸頂）；右下淺藍底被 `#14203C` 斜四邊形壓住、**左下「留出」淺藍三角（三角不是畫出來的，是被深藍壓出來的負空間）**；最右下 `#1C3667` 直欄。較有設計感。

**⚠️ 右側幾何一律「打開 `templates/cover-a.html`（或 `cover-b.html`）、把裡面的 `<svg>` 整段原封不動貼過來」，不要自己用 div＋CSS 或憑文字描述重畫。** 那幾塊是精算過的座標，freehand 一定跑版（某塊放太大、散開、溢到版面中間、顏色跑掉）。**只改文字，別動幾何。**（SVG 只住模板一處＝單一來源，不會走鐘。）
**幾何驗收（封面做完必看；分款對照，用數字量、不用感覺）**：
- **封面 A**：五個形狀**排成一直條**貼齊右緣，由上而下＝橘三角→白條→淺藍圓→深藍半圓→橘半圓；三角上緣 y=0、橘半圓下緣 y≈720、直條左界 x≈1137。
- **封面 B**：整片拼塊佔**右半**，左界 x=763、貼齊上（y=0）/ 下（y=720）/ 右（x=1280）；中欄（x 763–938）上下兩大塊夾四條橫條；拱頂碰到上緣；淺藍三角在**左下**（直角在右上）。**⚠️ 別拿封面 A 的「一直條」標準驗 B——B 本來就是整片拼塊，不是條。**
- 兩款共通：**若出現散開、某塊超大、溢到版面中間、或顏色不對（例如三角變綠）＝跑版了，回去整段套模板 SVG，別自己喬。**

**封面幾何 pptxgenjs 照抄寫法（重建 .pptx 時用——PPTX / Keynote / Google 簡報都會走這步）**：
HTML 的 `<svg>` 帶不進 pptxgenjs，重建時封面幾何得用原生 shape 重畫——**這一步最容易跑版，下面兩段照抄、別自己換算座標**（前提＝「產出與轉檔」的統一版面：`pptx.defineLayout({name:"kn1920",width:26.667,height:15})`、`const u=px=>px/48;`，座標仍用 1280×720px 網格；半圓用 `pie` 拼，矩形已做 1px 重疊防接縫髮絲線）：

**標題排版**（兩種皆可）：① 大標在橘斜線上方、次標在斜線下方（斜線當分隔，正規做法）；② 斜線在上，大標＋次標都放斜線下方。
左側所有物件（斜線、大標、次標、logo）**一律齊左 96px**。斜線用 SVG 畫，讓**最左點落在 96px**（別用旋轉方塊，旋轉後左角會凸出而不齊）。
**大標與斜線間距**：大標（尤其第二行）別貼著橘斜線。太擠時兩種解法擇一——① 若斜線下方不會太擠，把**斜線那組（斜線＋次標）往下挪**、開出空間；② 不動斜線以下，只**調整標題行距 / 位置**縮出間距。

## 書寫規範（產出文案一律遵守）

1. 中文字、中文標點一律**全形**。
2. 唯獨**括號用半形 `( )`、刪節號用半形 `...`**；半形括號前後加**半形空格**。
3. 英文、數字、英文標點一律**半形**；英文標點後加半形空格。
4. 中英文接續處加**半形空格**。
5. 品牌名：`AlleyPin`、`AlleyPin Nova`（產品全名 `AlleyPin Nova 企業卡`）、`PinMed`（兩字不空格、首字大寫）、`DBC`（全大寫）、`1.Talk`。
6. 第三方用對外全稱：`LINE`（非 Line）、`Facebook`（非 FB）、`Visa`（非 VISA）、`遠東商業銀行`（非 遠東商銀）。**全稱不確定先問使用者，勿臆測。**
7. 概數：成數用**國字**（如「五成」）；百分比用**半形數字**（如 `35%`）。
8. 金額：圓整大額用「N 萬」（半形數字＋國字單位，如 `10 萬`、`100 萬`）；精確金額用完整數字（如 `NT$5,000`）。

範例：`AlleyPin 提供診所最便利的 LINE CRM 服務，串接全台超過 11 家病歷系統商 ( HIS )，支持全台超過九成的診所營運。`

**適配判斷（克制＝有目的，不亂塞）**：

- **並列少字的項目，各配一個同尺寸同風格的小 icon / 插圖很好**（整齊、幫掃描）；**大主插圖才一頁一張**。

**擺放與尺寸**：
- **icon**：小（約 28–32px），放卡片右上角或標題旁；深灰線稿配白卡剛好。
- **插圖**：放版面空白半邊當支撐視覺（呼應「主動留白→加支撐視覺」），別壓字。
- **IP 角色**：當 hero。**讓角色面向 / 指向文案**（背對的用水平翻轉 `flipH`）；跟卡片 / 色塊 / band 有互動時，**對齊它的邊（如齊右緣）＋讓角色略壓進下緣**（避開文字區），比讓角色浮在旁邊留一條小縫更整齊。
- 一頁風格別混搭（icon 歸 icon、插圖歸插圖）。

**可編輯性**：PNG 以「圖片物件」置入（跟 logo 一樣可移動縮放）——**不違反禁截圖**（禁的是整頁截圖，不是正常置入圖片元件）。

**pptxgenjs 置入**（讀 PNG 標頭等比、不變形）：
```js
const fs=require('fs');
function pngSize(p){const b=fs.readFileSync(p);return{w:b.readUInt32BE(16),h:b.readUInt32BE(20)};}
// 依高度等比置入；cx=以 x 為中心；flipH=水平翻轉（讓 IP 面向文案）
function imgH(s,path,x,y,h,cx,flipH){const d=pngSize(path),w=h*d.w/d.h;
  s.addImage({path,x:u(cx?x-w/2:x),y:u(y),w:u(w),h:u(h),flipH:!!flipH});return w;}
```
HTML 模板則用 `<img src="assets/...">`。

**擴充 / 更新素材**：跑 `bash assets/sync-visual-assets.sh`——從 social-post-generator 同步並重建 `ASSET-INDEX.md`（維護者用）。

## Logo

`assets/alleypin_logo_white.png`（深藍底用）、`assets/alleypin_logo_blue.png`（淺底用），皆去背。**尺寸 855 × 210，比例約 4.07:1**。
footer 的「AlleyPin」用文字即可（Noto Sans TC）；封面左下、封底 lockup 用 logo 圖。內容頁通常不放 logo。
**⚠️ 置入務必等比、別硬寫不合比例的 w×h（會被拉扁 / 拉長，PPTX / Keynote / Google 簡報都會）**：HTML 只設**單邊**（`width` 或 `height`，另一邊 auto，模板已這樣）；pptxgenjs 給寬度、**`高 = 寬 / 4.07`**，或用 `pngSize` 讀原圖等比。任何圖片（icon / illus / IP）同理。

**pptxgenjs 安全寫法（照抄，別自己填兩個數字）**：
```js
const R=4.071;                       // logo 原始比例 855/210
const H=42;                          // 想要的高度(px)，只決定「一個」數字
s.addImage({path:LOGO, x:u(96), y:u(620), h:u(H), w:u(H*R)});   // 寬一律由比例算
// 最保險：直接用素材段的 imgH（讀 PNG 原尺寸等比）→ imgH(s,LOGO,96,620,42)
```
**只准決定寬或高其中一個，另一個永遠用比例／讀檔算出來。** 一旦你手上同時出現兩個寫死的數字，就是變形的開始。

## 產出與轉檔

> **🔒 硬性要求：交付檔一律「可編輯」，絕不接受截圖 / 圖片版。**
> 每一頁的文字、色塊、圖表、線條都必須是**原生可編輯物件**（PPTX / Keynote / Google 簡報裡點得到、改得動的 text box 與 shape）。
> **嚴禁**把 HTML 或某頁「截圖成 PNG 再貼進投影片」當作交付——那會讓使用者完全無法修改，是不合格的產出。
> 需要用程式產生 PPTX / Keynote 時，一律走 **pptxgenjs 依模板座標重建原生物件**這條路（見下第 3–4 點）；截圖只可用於「自己回頭檢查版面」，永遠不進交付檔。
>
> **🧭 單一來源：三種輸出格式的可編輯檔，全部源自同一個 pptxgenjs 重建的 `.pptx`。**
> - **PPTX**：直接就是這個 `.pptx`。
> - **Keynote**：用 Keynote 開這個 `.pptx` → 另存 `.key`（原生可編輯）。
> - **Google 簡報**：把這個 `.pptx` 上傳 Google Drive → 用「Google 簡報開啟」，Drive 會自動轉成**可編輯的原生 Google 簡報**（Chat App 的「open in Drive」即走此路）。
>
> **HTML 只是設計稿（給瀏覽器預覽、當座標參考），不是交付檔**——尤其 **HTML 無法直接變成可編輯的 Google 簡報**，別把 HTML 當 Slides 交付。
>
> **📐 統一版面（讓 Keynote 顯示值＝規範值）**：pptxgenjs 一律 `defineLayout({name:"kn1920",width:26.667,height:15})` ＋ `const u=px=>px/48;`——座標照 1280×720px 網格寫、**字級 `fontSize = px × 1.5`（＝規範表的 pt 值）**、行距 `lineSpacingMultiple` 直接給 1.1–1.3。這樣 Keynote 開啟顯示 **1920×1080**，檢閱器裡的字級、行距數字與本規範一字不差（2026-07-09 實測）。Google 簡報理論上照常吃這個尺寸，但**改版後第一份 Google 簡報交付要真的上傳 Drive 開起來確認版面與字級**。
>
> **⚖️ 可編輯是「門檻」，不是「終點」。** 交付前一律再過「收尾自檢」（對齊、重疊 / 溢出、層次、主動留白）＋核對品牌（色盤、字體、框架、書寫規範）。**可編輯但版面亂 / 走調的檔，一樣不合格**——品質與可編輯要同時成立。轉檔後（尤其 Keynote / Google 簡報）務必逐頁再看一遍，形狀 / 字距偶爾會微跑。

1. 複製對應模板**連同 `assets/` 資料夾**到你的簡報資料夾（logo 路徑用 `assets/...`）→ 填字（遵守書寫規範）→ 內容版面依主題設計 → 瀏覽器確認 1280×720。
2. 轉 **Google 簡報**：先照第 3 點產出**可編輯 `.pptx`** → 上傳 Google Drive → 右鍵「用 Google 簡報開啟」（或 Chat App 的「open in Drive」），Drive 自動轉成**原生可編輯的 Google 簡報**。標題請用 `Noto Sans TC`（LINE Seed 不在 Google 字體庫、會被替換）。**不要**期待「HTML 直接變 Google 簡報」——那條路產不出可編輯檔。
3. 轉 PPTX：可用 pptxgenjs 依模板座標重建；**封面幾何一律用「封面」段的 pptxgenjs 照抄 block，別自己把 SVG 換算成 shape**。要用 LINE Seed 須先確保字體已安裝或嵌入。
4. 轉 **Keynote（Mac）**：先照第 3 點產出**可編輯 `.pptx`** → 用 **Keynote 開啟該 .pptx**（會自動匯入成原生、可編輯的 Keynote 物件）→「檔案 → 儲存」成 `.key`。**Keynote 不吃 HTML，一律走 pptx 匯入；絕不是把 HTML 截圖貼進 Keynote。** 本機有裝 LINE Seed 標題就正確，沒裝改用 Noto。匯入後照「收尾自檢」再看一遍（少數形狀 / 字距可能微跑）。
   - 驗收：在 Keynote 隨手雙擊一段文字，**能進入編輯游標**＝合格；若整頁選起來是一張圖＝不合格，退回用 pptxgenjs 重做。

## 常見錯誤

- 標題用 LINE Seed 卻輸出 Google 簡報 → 字會掉。先問輸出格式。
- **LINE Seed 明明裝了卻沒生效** → CSS 家族名要用 `LINE Seed TW_OTF` / `LINE Seed TW_TTF`（不是 `LINE Seed TW`），否則靜默 fallback 到 Noto。用 `system_profiler SPFontsDataType | grep -i "line seed"` 查真名。
- 斜線用旋轉方塊 → 左緣不齊。用 SVG 線、最左點落 96px。
- **封面右側幾何跑版 / 散開 / 某塊超大 / 溢到中間 / 顏色不對**（同事實際踩過）→ 沒用模板的 `<svg>`，改用 div＋CSS 或憑文字描述重畫，複雜的絕對定位一定跑。**HTML 階段：整段複製 `cover-a` / `cover-b` 的 `<svg>`、只改文字；pptx 重建階段：用「封面」段的照抄 block。** 做完照**該款封面**的「幾何驗收」對（A 一直條五形狀；B 右半整片拼塊——別拿 A 的標準驗 B）。
- 左上矩形留了上邊距 → 要貼齊最上緣。
- 文案寫 `VISA` / `FB` / 全形括號 → 違反書寫規範。
- logo 不顯示 → 忘了把 `assets/` 一起複製、或 src 路徑沒指對。
- 為了套用範例版面硬湊內容（非數字主題硬塞大數字）→ 版面服務內容，不是反過來。
- 淺色當小內文 → 低對比、會糊。
- 封底收尾標題太長 / 換行 → 擠到聯絡人。標題要簡短（1 行）。
- **交付「截圖 / 圖片版」的投影片 → 嚴重錯誤**，使用者完全無法修改。任何 PPTX / Keynote / Google 簡報都必須是原生可編輯物件（見「產出與轉檔」硬性要求）；用 pptxgenjs 重建，別截圖。
- **pptxgenjs 畫斜線（關係圖連接線）畫反了** → pptx 的 `line` 是「bounding box 左上→右下」，斜線要自己判斷方向：當「衛星在中心右邊」與「衛星在中心下方」恰好一真一假時，設 `flipV:true`。公式 `flipV = ((ax>cx) !== (ay>cy))`（a=端點、c=中心）。HTML/SVG 沒這問題，直接 `<line>` 即可。
- **pptxgenjs 圓形排列的箭頭方向不一致**（循環圖 / 飛輪 / 環狀流程）→ `triangle` 預設頂點朝上，要沿圓形做「順時針切線」時，`rotate = 該點角度 + 180`（不是 +90）。角度以圓心為準（`-90` = 正上方）。寫錯會變成箭頭朝外、方向亂掉，尤其底部最明顯。
- **CSS `line-height` 直接抄進 pptxgenjs `lineSpacingMultiple`** → Keynote 行距虛胖約 1.46 倍（顯示 1.5+、版面鬆散）。規範值 1.1–1.3 直接給，別搬 CSS 數字（見「字級與行距」）。
- **中文內文用到 24pt（16px）以下** → 演講投影看不清。下限 24pt；只有免責 / 出處 / 頁碼等 chrome 例外。
- **數字 / 標籤文字框太窄 → 被換行**（如千分位 `1,050` 斷成兩行、軸標籤擠掉）。給數字與標籤留足寬度，或縮字級；tabular 數字尤其要留。
- **卡片放 icon 和標題 / 標籤重疊** → icon 放右上角、標題那側留出空間；icon 小（約 28–32px），別跟文字搶位。
- **素材放太大會糊** → icon / illus / IP 是點陣 PNG，別放大超過原尺寸太多：icon 小用、IP 別撐超過原寬。需要很大張的主視覺就挑本來就大的 IP 檔。
- **logo / 圖片被拉伸變形**（同事實際踩過，Google 簡報上最明顯）→ 指定的 w×h 比例跟原圖不符就會被拉扁 / 拉長。**logo 比例 4.07:1**（855×210）。HTML 只給單邊、另一邊 auto；pptxgenjs 給寬、`高 = 寬 / 4.07`，或用 `pngSize` 等比。放完**在實際輸出格式上再看一眼**。
